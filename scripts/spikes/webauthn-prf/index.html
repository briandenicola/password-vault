🔐 WebAuthn PRF Spike

Throwaway validation for the OFF-4 zero-knowledge design (docs/design/e2ee.md). Proves on this device that a passkey can derive an encryption key via the PRF extension and that the envelope flow (PRF → HKDF → KEK → wrap/unwrap a random DEK → AES-GCM encrypt/decrypt) works. Nothing here talks to a server or stores real secrets.

Environment
Step 1 — Enroll a passkey (enables the PRF extension)

Use a platform passkey (Face ID / Windows Hello / Touch ID) or a security key.

Step 2 — Set up the vault key (PRF → HKDF → KEK → wrap a random DEK; encrypt a sample)
Step 3 — Lock & unlock (re-derive KEK from the passkey, unwrap the DEK, decrypt the sample)
Verdict
Run steps 1–3.
